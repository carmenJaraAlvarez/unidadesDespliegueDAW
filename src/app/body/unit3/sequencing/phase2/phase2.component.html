Fase de Desarrollo: Conocer, comprender, pensar, practicar y actuar

¿Qué puede fallar?
descriptionDescripción: A partir de una aplicación web básica, analizar qué partes pueden suponer un riesgo de seguridad: formularios, gestión de usuarios, contraseñas y acceso a la base de datos. Se trabaja la idea de que no todas las partes de una aplicación tienen el mismo nivel de riesgo.
menu_bookRecursos: Esquema visual de una aplicación web, ejemplos reales de formularios y código sencillo.
groupsAgrupamiento: Asamblea y trabajo en pequeños grupos.
¿Qué podemos hacer? Seamos estoicos
descriptionDescripción: Reacción vs prevención. Se pretende realizar aquí una colaboración conjunta al departamento de Filosofía del centro si existiese o de otro centro. Se incluirá elementos como Premeditatio Malorum y dicotomía del control.
Actividades prácticas sobre autenticación
descriptionDescripción: Actividades prácticas sobre implementación de autenticación como elemento habitual. El alumnado añade a su aplicación un sistema sencillo de usuarios con registro e inicio de sesión. Se trabaja la diferencia entre estar identificado y tener permisos dentro de una aplicación.
menu_bookRecursos: Código base proporcionado por el profesorado, tutorial guiado paso a paso.
groupsAgrupamiento: Trabajo en grupos.
Cifrado de contraseñas
descriptionDescripción: El alumnado modifica el sistema de autenticación para que las contraseñas no se almacenen en texto plano. Se explica de forma sencilla por qué esto es necesario y qué problemas puede causar no hacerlo.
menu_bookRecursos: Ejemplos comparativos (contraseña sin cifrar / contraseña cifrada), esquemas visuales.
groupsAgrupamiento: Grupos.
Detección de vulnerabilidades

  • Instalación de plugin Snyk en el IDE (herramienta profesional que podemos usar si nos registramos)
  • Análisis de resultados
